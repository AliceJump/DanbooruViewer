# 拖拽功能改进 - 视觉反馈优化

## 问题描述
在之前的版本中，用户长按图片进行拖拽时，只能看到系统默认的图标，看不到被拖拽的图片本身，导致视觉反馈不清晰。

## 解决方案

### 1. Android 原生端改进 (MainActivity.kt)

#### 新增 ImageDragShadowBuilder 类
- **功能**: 自定义拖拽阴影 Builder，在拖拽时显示图片缩略图
- **关键特性**:
  - 从文件路径加载图片
  - 自动缩放图片到 200x200 像素以优化性能
  - 使用 BitmapFactory 的 inSampleSize 降采样以减少内存占用
  - 在拖拽时绘制实际的图片缩略图而不是默认图标

#### 改进 startDragDrop 方法
- 使用新的 `ImageDragShadowBuilder` 替换默认的 `View.DragShadowBuilder`
- 保持了原有的拖拽标志和 FileProvider 集成

### 2. Flutter 端改进 (post_detail_page.dart)

#### 增强 GestureDetector
添加了以下回调以改进用户交互体验：
- `onLongPressStart`: 长按开始时的反馈
- `onLongPressEnd`: 长按结束时的清理
- `onLongPressMoveUpdate`: 追踪拖拽位置变化
- 保留原有的 `onLongPress` 用于启动实际拖拽

#### 状态管理
- 更新 `_isLongPressing` 标志
- 更新 `_currentDragOffset` 记录拖拽位置
- 准备为将来的高级功能提供基础（如方向感知的可视反馈）

## 改进结果

### 用户体验提升
1. **清晰的视觉反馈**: 用户现在能看到正在拖拽的图片缩略图
2. **更好的交互体验**: 拖拽时有明确的视觉指示
3. **性能优化**: 缩略图被限制在 200x200 像素，避免内存压力

### 技术优化
1. **内存管理**: 使用降采样加载图片，再进行缩放
2. **资源清理**: Bitmap 在使用后及时回收
3. **兼容性**: 保持与 Android 6.0+ 的兼容性

## 测试建议
1. 在图片详情页长按图片，观察拖拽阴影是否显示缩略图
2. 尝试拖拽到不同的应用进行分享
3. 检查内存占用是否合理
4. 验证不同尺寸的图片缩略图显示效果
